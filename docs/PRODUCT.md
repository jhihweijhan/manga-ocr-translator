# WebUI 圖片翻譯產品規格

## Summary

這個 WebUI 讓使用者上傳單張圖片，從本機 Ollama 模型清單中分別選擇 OCR 模型與翻譯模型，自動產生 OCR 原文與目標語言譯文。第一版以功能可用、結果可檢查、提示詞可觀察為主，不保存任務歷史，也不做文字區域偵測。

## Goals

1. 使用者可以用本機 Ollama 模型完成單張圖片的 OCR 與翻譯。
2. 使用者可以看到 OCR 原文、區塊譯文、使用中的模型與提示詞內容，以便判斷品質。
3. 使用者可以修正 OCR 原文後重新翻譯，不需要重跑 OCR。
4. 第一版保持單一任務、無持久保存、無批次、無嵌字回圖。

## Non-goals

1. 不支援多張圖片批次。
2. 不保存任務歷史，重新整理頁面會清空目前翻譯任務。
3. 不匯入先前匯出的 JSON。
4. 不保存上傳圖片，也不匯出圖片檔本身。
5. 不做文字區域偵測模組；文字區塊可以沒有位置資訊。
6. 不做嵌字回圖片、TXT/CSV 匯出、PDF/GIF/CBZ/ZIP 處理。
7. 不在 WebUI 編輯或儲存提示詞設定。

## Figma

Figma: none provided. 第一版先以功能完整、狀態清楚、可檢查結果為準。

## Behavior

1. 進入 WebUI 時，畫面顯示 Ollama 位址設定，預設為 `http://127.0.0.1:11434`，使用者可以修改。

2. WebUI 載入後會自動讀取 Ollama 模型清單，並提供手動重新整理模型清單的控制。

3. 模型清單讀取失敗時，畫面顯示連線或讀取錯誤。使用者仍可上傳圖片並看到圖片預覽，但在成功載入模型並完成模型選擇前，不能開始翻譯任務。

4. OCR 模型與翻譯模型是兩個獨立下拉選單。兩者都從同一份 Ollama 本機模型清單載入。

5. 第一版不自動判斷模型能力，也不根據模型清單過濾 vision 或翻譯能力。若使用者選到不適合的模型，該階段會以錯誤狀態呈現。

6. 第一次使用時，OCR 模型與翻譯模型不自動猜選。之後 WebUI 會記住上次選擇的 Ollama 位址、OCR 模型、翻譯模型、來源語言提示、目標語言與逾時設定；若模型仍存在於清單中，會自動帶入。

7. 使用者可以上傳一張 PNG、JPEG 或 WebP 圖片。單張圖片大小上限為 10 MB，超過時在前端阻擋並顯示錯誤。

8. WebUI 同一時間只處理一個翻譯任務。上傳新圖片會取代目前頁面中的翻譯任務狀態與結果。

9. 翻譯任務的來源語言提示預設為 `自動判斷`。第一版可選值為 `自動判斷`、`日文`、`英文`、`韓文`、`簡體中文`、`繁體中文`。來源語言提示只是提供給模型的線索，不代表模型一定支援或遵守。

10. 翻譯任務的目標語言預設為 `繁體中文`。第一版可選值為 `繁體中文`、`簡體中文`、`英文`、`日文`、`韓文`。

11. 使用者上傳圖片並選定 OCR 模型與翻譯模型後，WebUI 會自動連續執行 OCR 與翻譯，不需要使用者在 OCR 後再按一次翻譯。若使用者先上傳圖片但尚未選定必要模型，WebUI 會等待使用者補齊模型選擇後自動處理一次。完成後再更改設定不會自動重跑，必須由使用者按「重新處理」或「重新翻譯」。

12. OCR 階段完成後會產生文字區塊。每個文字區塊至少包含 OCR 原文；位置資訊不是必要條件。

13. 翻譯階段會針對目前所有文字區塊產生一對一的區塊譯文。每個區塊譯文都必須用 `block_id` 對應一個文字區塊，顯示順序永遠依原始文字區塊順序。

14. 結果區以上方圖片預覽搭配下方左右對照表呈現。每列代表一個文字區塊，左欄是可編輯 OCR 原文，右欄是目標語言譯文。

15. 使用者修改 OCR 原文後，不會自動重新翻譯。使用者必須按「重新翻譯」，WebUI 才使用目前文字區塊重新產生區塊譯文。

16. 「重新翻譯」只重跑翻譯階段，不重跑 OCR。若翻譯成功，新的區塊譯文取代目前譯文。

17. 「重新處理」會重跑 OCR 與翻譯，並以新的文字區塊與區塊譯文取代目前結果。每次處理都有唯一 `run_id`；舊請求回應不得覆蓋目前 `run_id` 的任務結果。

18. 使用者更換 OCR 模型、翻譯模型、來源語言提示、目標語言或 Ollama 位址後，可以重新處理目前圖片。目標語言變更後主要動作是「重新翻譯」；來源語言提示同時影響 OCR 與翻譯，使用者可以依需要選擇「重新處理」或「重新翻譯」。

19. OCR 或翻譯進行中，模型選單與會影響請求的設定會被鎖定。使用者若要變更設定，必須先取消處理或等待目前階段完成。

20. OCR 或翻譯進行中，畫面提供取消處理。取消 OCR 會保留圖片預覽、模型選擇與設定，並清空文字區塊與區塊譯文。取消翻譯會保留已完成的文字區塊並清空未完成的區塊譯文。取消後舊請求回應不得覆蓋目前畫面。

21. OCR 失敗時，翻譯階段不會開始。畫面保留圖片預覽與 OCR 錯誤，使用者可以更換 OCR 模型或設定後重新處理。

22. OCR 成功但翻譯失敗時，畫面保留 OCR 文字區塊，標示翻譯失敗。使用者可以更換翻譯模型、修正原文或調整設定後重新翻譯。

23. OCR 成功但沒有任何文字區塊時，不呼叫翻譯階段。任務進入完成狀態，畫面顯示「未偵測到可翻譯文字」，匯出 JSON 包含空的文字區塊與區塊譯文。

24. 若模型回傳的 JSON 不符合預期格式，該階段視為失敗。OCR JSON 不合法時不進入翻譯；翻譯 JSON 不合法時保留 OCR 結果。

25. 模型回應不做串流顯示。WebUI 等待完整回應可驗證後，才更新 OCR 或翻譯結果。

26. OCR 與翻譯請求有可修改逾時設定，預設為 120 秒。逾時時，該階段視為失敗並顯示逾時錯誤。

27. WebUI 會顯示目前有效的 OCR 與翻譯提示詞模板。提示詞來源可能是外部 `prompts.toml` 或內建預設，畫面必須清楚標示來源。每次 OCR 或翻譯結果與匯出 JSON 會保存該次請求實際送出的 rendered prompt。

28. WebUI 顯示提示詞時只讀，不提供編輯與儲存。

29. 提示詞設定檔名固定為 `prompts.toml`。若 `prompts.toml` 不存在，系統可使用內建預設提示詞，且 WebUI 必須顯示內建預設提示詞內容。

30. 若 `prompts.toml` 格式錯誤，相關請求失敗並顯示 TOML 錯誤，不靜默改用其他提示詞。

31. 完成後，使用者可以匯出 JSON。匯出內容包含圖片檔名、OCR 模型、翻譯模型、來源語言提示、目標語言、文字區塊、區塊譯文、提示詞來源、提示詞模板與 rendered prompt。

32. 匯出 JSON 不包含圖片檔本身。

33. 頁面狀態至少包含 `idle`、`ready`、`ocr_running`、`ocr_failed`、`translation_running`、`translation_failed`、`completed`、`cancelled`。每個狀態都應讓使用者看得出目前能做什麼、不能做什麼，以及錯誤發生在哪個階段。
