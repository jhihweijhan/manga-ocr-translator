# WebUI 圖片翻譯產品規格

## Summary

這個 WebUI 目前讓使用者上傳單張圖片，從本機 Ollama 模型清單中分別選擇 OCR 模型與翻譯模型，自動產生 OCR 原文與目標語言譯文。第一版已實作主流程以功能可用、結果可檢查、提示詞可觀察為主，不保存任務歷史，並維持 ADR 0004 的完整回應處理方式。文字區域 overlay、串流進度、多頁閱讀與 CBZ/ZIP 仍是 ADR 0010、0011、0012 的 proposed design-first 內容，尚未實作。

## Goals

1. 使用者可以用本機 Ollama 模型完成單張圖片的 OCR 與翻譯。
2. 使用者可以看到 OCR 原文、區塊譯文、使用中的模型與提示詞內容，以便判斷品質。
3. 使用者可以修正 OCR 原文後重新翻譯，不需要重跑 OCR。
4. 第一版保持單一閱讀任務、無持久保存、無多任務並行、無嵌字回圖。

## Non-goals

1. 不支援多個閱讀任務並行、背景佇列或任務歷史。
2. 不保存任務歷史，重新整理頁面會清空目前翻譯任務。
3. 匯入先前匯出的 JSON 時不還原圖片本體；只還原文字區塊、譯文、設定與提示詞紀錄。
4. 不保存上傳圖片，也不匯出圖片檔本身。
5. 依 ADR 0010，文字區域 overlay 與 OCR `position` normalized rectangle 支援尚未實作；現行程式可維持 `position: null`，清單模式仍是相容且有效的主流程。
6. 依 ADR 0011，串流進度尚未實作；現行主流程仍使用 `/api/ocr` 與 `/api/translate` 完整回應（ADR 0004）。
7. 依 ADR 0012，多頁閱讀、資料夾與 CBZ/ZIP archive 尚未實作；CBZ/ZIP 在落地前維持 non-goal/proposed。
8. 不做嵌字回圖片、CSV 匯出、PDF/GIF 處理。
9. 不在 WebUI 編輯或儲存提示詞設定。

## Figma

Figma: none provided. 第一版先以功能完整、狀態清楚、可檢查結果為準。

## Behavior

1. 進入 WebUI 時，畫面顯示 Ollama 位址設定，預設為 `http://127.0.0.1:11434`，使用者可以修改。

2. WebUI 載入後會自動讀取 Ollama 模型清單，並提供手動重新整理模型清單的控制。

3. 模型清單讀取失敗時，畫面顯示連線或讀取錯誤。使用者仍可上傳圖片並看到圖片預覽，但在成功載入模型並完成模型選擇前，不能開始翻譯任務。

4. OCR 模型與翻譯模型是兩個獨立下拉選單。兩者都從同一份 Ollama 本機模型清單載入。

5. 第一版不自動判斷模型能力，也不根據模型清單過濾 vision 或翻譯能力。若使用者選到不適合的模型，該階段會以錯誤狀態呈現。

6. 第一次使用時，OCR 模型與翻譯模型不自動猜選。之後 WebUI 會記住上次選擇的 Ollama 位址、OCR 模型、翻譯模型、來源語言提示、目標語言與逾時設定；若模型仍存在於清單中，會自動帶入。

7. 現行 WebUI 支援上傳單張 PNG、JPEG 或 WebP 圖片。單張圖片大小上限為 10 MB，超過時在前端阻擋並顯示錯誤。

8. 依 ADR 0012，尚未實作的 proposed 多頁來源會被正規化成頁清單。多檔輸入依檔名排序；資料夾、CBZ 與 ZIP 依正規化相對路徑排序。排序採穩定 natural sort，數字片段以數值比較；忽略 `__MACOSX`、隱藏檔與隱藏目錄，只接受 PNG、JPEG 與 WebP 頁面。CBZ/ZIP 在實作前不是現行行為。

9. WebUI 同一時間只處理一個任務。現行上傳新圖片會取代目前任務狀態與結果。依 ADR 0012，尚未實作的多頁重新連結圖片來源流程會只把新選來源的圖片 Blob/object URL 接回既有頁面，不取代既有 OCR、譯文、頁狀態或提示詞紀錄。

10. 翻譯任務的來源語言提示預設為 `自動判斷`。第一版可選值為 `自動判斷`、`日文`、`英文`、`韓文`、`簡體中文`、`繁體中文`。來源語言提示只是提供給模型的線索，不代表模型一定支援或遵守。

11. 翻譯任務的目標語言預設為 `繁體中文`。第一版可選值為 `繁體中文`、`簡體中文`、`英文`、`日文`、`韓文`。

12. 使用者上傳圖片並選定 OCR 模型與翻譯模型後，WebUI 會自動連續執行 OCR 與翻譯，不需要使用者在 OCR 後再按一次翻譯。若使用者先上傳圖片但尚未選定必要模型，WebUI 會等待使用者補齊模型選擇後自動處理一次。完成後再更改設定不會自動重跑，必須由使用者按「重新處理」或「重新翻譯」。多頁來源與「重跑此頁」屬 ADR 0012 proposed，尚未實作。

13. 依 ADR 0012，尚未實作的 proposed 多頁任務會以逐頁佇列處理，第一版同一時間最多處理一頁。每頁先執行 OCR，再執行該頁翻譯；下一頁必須等目前頁的 OCR 與翻譯完成或失敗後才開始。取消目前頁會暫停佇列，不會自動開始下一頁。

14. OCR 階段完成後會產生文字區塊。每個文字區塊至少包含 OCR 原文；位置資訊不是必要條件。

15. 翻譯階段會針對目前圖片的所有文字區塊產生一對一的區塊譯文。每個區塊譯文都必須用 `block_id` 對應同一張圖片的一個文字區塊，顯示順序永遠依原始文字區塊順序。依 ADR 0012，尚未實作的多頁匯出會維持 page-local `block_id`，只有前端內部全域查找、搜尋或未來全書視圖需要跨頁引用時，才使用 `page_id` 加上頁內 `block_id` 的命名空間。

16. 現行結果區以單張圖片預覽與左右對照表呈現，每列代表一個文字區塊，左欄是可編輯 OCR 原文，右欄是目標語言譯文。依 ADR 0012，頁清單、縮圖列與頁間導航尚未實作。

17. 使用者修改 OCR 原文後，不會自動重新翻譯。現行使用者必須按「重新翻譯」，WebUI 才使用目前文字區塊重新產生區塊譯文。依 ADR 0012，「重跑此頁翻譯」屬多頁 proposed，尚未實作。

18. 「重新翻譯」只重跑翻譯階段，不重跑 OCR。現行單張圖片流程只取代目前圖片譯文。依 ADR 0012，整個多頁任務依頁清單順序逐頁重跑翻譯尚未實作。

19. 「重新處理」會重跑 OCR 與翻譯，並以新的文字區塊與區塊譯文取代目前圖片結果。每次處理都有唯一 `run_id`；舊請求回應不得覆蓋目前 `run_id` 的任務結果。多頁目標範圍重跑屬 ADR 0012 proposed，尚未實作。

20. 使用者更換 OCR 模型、翻譯模型、來源語言提示、目標語言或 Ollama 位址後，可以重新處理目前單張圖片任務。目標語言變更後主要動作是「重新翻譯」；來源語言提示同時影響 OCR 與翻譯，使用者可以依需要選擇「重新處理」或「重新翻譯」。依 ADR 0012，目前頁/整本任務範圍選擇屬多頁 proposed，尚未實作。

21. OCR 或翻譯進行中，模型選單與會影響請求的設定會被鎖定。使用者若要變更設定，必須先取消處理或等待目前單張圖片流程完成。依 ADR 0012，等待目前頁完成屬多頁 proposed，尚未實作。

22. OCR 或翻譯進行中，畫面提供取消處理。現行取消 OCR 會保留圖片預覽、模型選擇與設定，並清空文字區塊與區塊譯文。取消翻譯會保留已完成的文字區塊並清空未完成的區塊譯文。取消後舊請求回應不得覆蓋目前畫面。依 ADR 0012，取消整個佇列、取消單頁、重跑取消頁、跳過該頁後繼續佇列尚未實作。

23. 現行 OCR 失敗時，翻譯階段不會開始，畫面保留圖片預覽與 OCR 錯誤。使用者可以更換 OCR 模型或設定後重新處理。依 ADR 0012，多頁失敗隔離、重跑單頁與繼續其他頁尚未實作。

24. 現行 OCR 成功但翻譯失敗時，畫面保留 OCR 文字區塊，標示翻譯失敗。使用者可以更換翻譯模型、修正原文或調整設定後重新翻譯。依 ADR 0012，重跑單頁翻譯尚未實作。

25. 現行 OCR 成功但沒有任何文字區塊時，不呼叫翻譯階段，畫面顯示「未偵測到可翻譯文字」，匯出 JSON 包含空的文字區塊與區塊譯文。依 ADR 0012，頁狀態尚未實作。

26. 若模型回傳的 JSON 不符合預期格式，該階段視為失敗。現行 OCR JSON 不合法時不進入翻譯；翻譯 JSON 不合法時保留 OCR 結果。依 ADR 0012，逐頁錯誤歸屬屬多頁 proposed，尚未實作。

27. 依 ADR 0011，尚未實作的 proposed 串流進度可在 OCR 與翻譯進行中顯示模型處理進度，例如目前階段、已收到的模型片段數、已耗時或「正在產生回應」等非權威訊息。進度只用來降低長任務等待的不確定感，不代表 OCR 文字區塊或譯文已可用。現行主流程仍是 `/api/ocr`、`/api/translate` 完整回應（ADR 0004）。

28. 模型內容本身不做逐字串流顯示，也不顯示未驗證的 partial JSON。現行與 ADR 0011 proposed 流程都必須等待後端取得完整 Ollama 回應並通過 schema 驗證後，才更新 OCR 文字區塊或翻譯結果。

29. 依 ADR 0011，尚未實作的 proposed 進度串流若以最終結果結束，WebUI 會進入下一階段或完成狀態；若以錯誤結束，WebUI 依錯誤階段進入 `ocr_failed` 或 `translation_failed`。最終錯誤必須使用與一般 API 相同的錯誤 envelope，讓畫面可一致呈現。多頁下一頁轉換屬 ADR 0012 proposed，尚未實作。

30. OCR 與翻譯請求有可修改逾時設定，預設為 120 秒。逾時時，現行階段視為失敗並顯示逾時錯誤。依 ADR 0012，逐頁逾時歸屬屬多頁 proposed，尚未實作。

31. WebUI 會顯示目前有效的 OCR 與翻譯提示詞模板。提示詞來源可能是外部 `prompts.toml` 或內建預設，畫面必須清楚標示來源。每次 OCR 或翻譯結果與匯出 JSON 會保存該次請求實際送出的 rendered prompt。

32. WebUI 顯示提示詞時只讀，不提供編輯與儲存。

33. 提示詞設定檔名固定為 `prompts.toml`。若 `prompts.toml` 不存在，系統可使用內建預設提示詞，且 WebUI 必須顯示內建預設提示詞內容。

34. 若 `prompts.toml` 格式錯誤，相關請求失敗並顯示 TOML 錯誤，不靜默改用其他提示詞。

35. 完成後，使用者可以匯出 JSON。現行單頁匯出使用 v1 結構。依 ADR 0012，尚未實作的多頁匯出會使用多頁結構，包含每頁 `page_id`、檔名、相對路徑、排序索引、頁狀態、OCR 模型、翻譯模型、來源語言提示、目標語言、文字區塊、區塊譯文、提示詞來源、提示詞模板與 rendered prompt。

36. 匯出 JSON 不包含圖片檔本身。

37. 完成後，使用者可以匯出純譯文 TXT。現行 TXT 內容依單張圖片的文字區塊順序輸出區塊譯文；沒有區塊譯文的文字區塊輸出為空行，不輸出「尚未翻譯」提示字。依 ADR 0012，跨頁依頁清單順序輸出尚未實作。

38. 前端保留原圖 Blob/object URL 僅限目前瀏覽器工作階段。重新整理頁面、關閉分頁或匯入不含圖片的 JSON 後，WebUI 可顯示已匯入的文字結果，但重新 OCR 或重新處理前必須重新選圖。依 ADR 0012，尚未實作的多頁「重新連結圖片來源」流程必須完整匹配既有頁清單：頁數相同，且每頁以 `relative_path` 優先、否則以 `filename` 加 `index` 對應；匹配失敗時不連結任何頁，也不覆蓋已匯入文字結果。

39. 依 ADR 0012，CBZ/ZIP archive 是 proposed 且尚未實作；在實作前維持 non-goal。若未來落地，必須有解壓安全限制，包含頁數上限、單頁大小上限、總解壓影像大小上限、壓縮比上限與路徑安全檢查。超出限制時拒絕整個 archive 並顯示可理解錯誤。

40. 任務狀態至少包含 `idle`、`ready`、`ocr_running`、`ocr_failed`、`translation_running`、`translation_failed`、`completed`、`cancelled`。依 ADR 0012，尚未實作的每頁狀態至少包含 `queued`、`ocr_running`、`ocr_failed`、`translation_running`、`translation_failed`、`completed`、`cancelled`。每個狀態都應讓使用者看得出目前能做什麼、不能做什麼，以及錯誤發生在哪個階段；多頁落地後也必須顯示錯誤發生在哪一頁。
